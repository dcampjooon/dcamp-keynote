# CLAUDE.md — dcamp-keynote (발표 키노트 제작 프로젝트)

상준님의 디캠프 사내 발표용 **애니메이션 HTML 키노트**를 만드는 프로젝트. 이 문서는 키노트를 만들 때 매번 기억해야 할 규칙과 학습을 누적하는 곳이다.

> **운영 규칙 1**: 앞으로 작업하며 "메모리에 담을 수준"의 중요한 발견·선호·결정이 생기면, 글로벌 메모리가 아니라 **이 파일에 누적**한다. 새 학습은 아래 해당 섹션에 한 줄씩 추가.
> **운영 규칙 2 (상시)**: 별도 지시가 없어도 **모든 작업 변경은 의미 단위로 git 커밋 + `origin/main` push**한다. (상준님 상시 지시, 2026-06-01) 원격: `git@github.com:dcampjooon/dcamp-keynote.git`. 커밋 메시지는 한국어 + `Co-Authored-By` 푸터.

## 1. 산출물 구조 / 작업 방식

- **산출물은 날짜별 폴더로 정리**한다. 예: `2026-05-31/index.html`, `2026-05-31/script.md`. 새 작업 세션은 그날 날짜 폴더를 새로 만들어 진행.
- 한 발표 = **원고(`script.md`) + 화면(`index.html`)** 두 파일. 원고를 먼저 확정하고 화면을 만든다.
- 화면은 **단일 HTML 파일**(외부 의존성은 Pretendard 폰트 CDN 하나). 더블클릭만으로 풀스크린 발표가 되게.
- 변경 후에는 **브라우저로 직접 열어 확인**한다. Windows에서 해시 URL은 `Invoke-Item`이 못 여니 `Start-Process "file:///<경로>/index.html#10"` 형태로 연다(일반은 `Invoke-Item` 가능).
- 슬라이드를 추가/삭제/이동하면: ① 하단 `pager`의 총 개수 ② 제거된 SVG를 채우던 `buildXxx()`에 `if(!컨테이너)return;` 가드 ③ SVG `id` 중복/누락 을 꼭 점검.

## 2. 발표 대상 / 톤 (가장 중요)

- **청중**: 디캠프 전 직원 — 비전공자가 다수. 기술 약어(ELT/RLS/SSOT/ADR 등)는 화면에 그대로 띄우지 말고 말로 풀거나 한국어로.
- **핵심 의사결정자(대표)**: 컨설팅펌 출신 → **다이어그램·플로우차트·전체 조망(top-down)·인사이트 도출** 형태를 선호.
- **컨설팅식 구성**: 각 슬라이드는 ① 헤드라인(결론·인사이트 한 줄) → ② 다이어그램(근거) → ③ 구두 설명. 헤드라인만 따라가도 논리가 이어지게.
- **헤드라인 규칙**: 상단 보조텍스트(kicker) 없이 **주 텍스트만, 중앙 정렬**. **명사형 종결 + 조사 최소화** (예: "→ 하나의 SSOT로 수렴", "한 단계 장애도 전체 무중단"). `→`·`·` 활용.
- **톤 다운**: 읍소조("우리 힘만으론 못한다/도와달라/고맙다") 금지. 기여·성과는 **사실·구조 진술**로 ("이 구조는 이미 작동하고 있다").
- **무게중심**: '계획'이 아니라 **실적·결과물·과정**. "할 겁니다"가 아니라 "이렇게 만들었습니다".

## 3. 시각화 원칙

- mermaid/정적 이미지를 그대로 넣지 말고 **SVG로 새로 그려 애니메이션**을 입힌다: 선이 그려지고(`stroke-dashoffset` + `pathLength=1`) 데이터 입자가 경로를 따라 흐르는(`<animateMotion>`+`<mpath>`) 연출. 이 연출이 이 키노트의 시그니처.
- **한 화면 밀도를 높게**. 2-컬럼(`split2`) 레이아웃으로 시각요소를 빽빽하게. 텍스트만 있는 슬라이드는 지양 — 표·타임라인·카드·다이어그램으로 시각화.
- 다이어그램은 JS로 동적 생성(`buildRelmap`/`buildFlow`/`buildBridge` 등). 노드 좌표 객체 + edge 배열 + flowdot 패턴.
- 색: 딥 네이비 배경 + 블루→시안 그라데이션 액센트. 출처/도메인별 색 구분(KODATA=초록, KiiPS=파랑, 노션=보라 등).
- "아주 많은데 일부만" 느낌은 **카드 다수 + `mask-image` 하단 페이드아웃**으로. (배경색 오버레이보다 mask가 자연스러움)
- `overflow:hidden` 영역에서 카드가 float(±px)하면 위가 잘린다 → **margin 말고 padding(상하)** 으로 여유를 준다.
- 애니메이션 엔진: 슬라이드 active마다 `[data-anim]`에 stagger로 `.show` 부여, 카운트업/게이지/타임라인 fill/SVG draw/입자/`.qcard` 등 재생.
- **SVG 레이어 순서는 반드시 `edges < dots < nodes`** (DOM 순서: Edges → Dots → Nodes). 점이 노드 위로 지나가면 어색 → 노드가 점을 가려 "노드 사이로만 점이 흐르는" 모습이 자연스럽다.
- **노드 배경은 불투명**으로(반투명이면 뒤의 연결선이 비쳐서 "선이 노드 위에 있는 것처럼" 보인다).
- **애니메이션 순서는 노드 먼저 등장 → 선이 따라 그려짐**(선이 먼저 나오면 "선 뭉치가 먼저 뜬 뒤 그림이 완성되는" 어색함). `edgeStart`를 노드 개수에 비례해 늦춘다.

## 4. 네비게이션 기능

- `←/→`·`Space`·클릭(좌16%/우)·`Home/End` 이동, `F` 풀스크린, 하단 dots 클릭 점프.
- **URL 해시 = 슬라이드 번호(1-indexed)**. `index.html#10`으로 열거나 새로고침하면 그 슬라이드로 바로. 이동 시 `history.replaceState`로 주소 자동 갱신(뒤로가기 히스토리는 안 더럽힘). `hashchange`로 주소창 직접 입력도 대응.

## 5. 디캠프 도메인 사실 (틀리면 안 됨)

- **통합DB = SSOT(Single Source of Truth)**. 화면엔 풀네임으로 1회 각인 후 `SSOT` 약어. "정본"이라는 한국어보다 **SSOT** 선호(상준님 결정).
- "포트폴리오사" ✗ → **패밀리사** / "프로그램팀" ✗ (디캠프에 없음). 배치 **선발=투자팀, 육성=Growth**.
- 7개 부서: 펀드 / 투자 / 그로스 / 글로벌 / 홍보마케팅 / 리소스 / 스페이스.
- **전사 AX 그림(2-Track)**: 현업 업무 시스템(접점, DX) + 통합DB(근간, SSOT) + AI(지능). 실행체계 = **AX Build**(IT팀 직접) + **AX Support**(현업이 직접 만들고 IT팀이 지원: 투자팀 AI 심사역, 사업실 AI 멘토, 경영본부 재무 대시보드, 전문자문 bd.dcamp.kr, 오피스아워 officehour.dcamp.kr). 성숙도: **AI-centric → AI-driven → 자기주도 AX**.
- **외주(팩트시트)**: AI 활용·개발 방법론·속도가 IT팀 기준에 못 미쳐 **계약 중도 정리 → 2026년 5월 자체+FDE 전환**. 비방 아닌 담백한 사실로 언급(상준님이 명시적으로 OK).
- 작업 여정: 2025말 ISP·입찰 RFP·IT멘토 자문("이 분량 어렵다") → 외주 → 자체+AI 전환 → **4~5주 만에 대부분 데이터 검증 완료**.
